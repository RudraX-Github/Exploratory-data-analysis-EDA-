Log #001 Transportation & Logistics

NYC Green Taxi Estimator

Python 3.9 XGBoost PyDeck 3D

The Challenge

New York City's transit system generates millions of data points daily, yet predicting accurate fares is plagued by volatility due to traffic congestion, borough-specific surcharges, and time-of-day variance. The goal was to build a model robust enough to predict fares within a 5% margin of error using only trip metadata.

The Solution

I engineered an ensemble pipeline centered around XGBoost, chosen for its gradient boosting capabilities on structured data. Feature engineering included extracting trip_distance (Haversine formula), temporal cyclical features (hour/day), and decoding complex RateCodeIDs. The front-end uses PyDeck to render 3D arc layers, visualizing high-traffic corridors in real-time.

Log #002 Media & Entertainment

CineMatch Engine

Scikit-Learn NLP Vectorization

The Challenge

Standard collaborative filtering fails with "Cold Start" problems for new users or niche content. The objective was to create a content-based recommendation system that understands the context of a film (plot, genre, cast) rather than just user ratings, ensuring 100% coverage of the database.

The Solution

I implemented a Vector Space Model using TF-IDF (Term Frequency-Inverse Document Frequency) to convert textual metadata into 5000-dimensional vectors. Recommendations are generated via Cosine Similarity matrices, measuring the angular distance between movies. The system was optimized by pre-computing the similarity matrix, reducing runtime complexity from O(N²) to O(1) for lookups.
